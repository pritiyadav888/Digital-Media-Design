Week 3: JavaScript & Web Interactivity

Welcome to Week 3! This module is all about JavaScript, the programming language that brings your web pages to life. You'll learn how to make your sites interactive, dynamic, and responsive to user actions. We'll cover everything from basic JavaScript concepts to manipulating the Document Object Model (DOM) and preparing your projects for the web.

By the end of this week, you'll understand how to add behavior to your HTML and CSS, transforming static content into engaging web experiences.

Why JavaScript?

JavaScript is the "brain and reflexes" of your webpage. While HTML provides the structure (skeleton) and CSS adds the style (skin), JavaScript allows your page to be interactive and dynamic. It makes things happen!

HTML can only display content. You write `<p>Hello</p>`, and it's there. But it cannot change once the page is loaded.

Want something to change based on the user's action (e.g., click a button and change text)? HTML can’t do that.

That's why we use JavaScript — it adds behavior.

Example (HTML only):

<button>Hello</button>
<p>This is always here.</p>

Example (With JavaScript):

<button onclick="sayHello()">Click Me</button>
<p id="message"></p>
<script>
  function sayHello() {
    document.getElementById("message").innerText = "Hello, Web Wizard!";
  }
</script>

JavaScript Basics

Let's break down some fundamental JavaScript concepts using real-life analogies to make them easier to grasp.

`let name = "Priti";` → Think of a labelled jar with your name in it.

  • `let`: Keyword to declare a variable (a container for a value).
  • `name`: The name of the variable (the label on the jar).
  • `=`: Assignment operator (puts the value on the right into the variable on the left).
  • `"Priti"`: The value being stored (content inside the jar), a string of text.

`function` → Like a recipe. It runs when you tell it to.

  • `function`: Keyword to define a block of reusable code.
  • The name after `function` (e.g., `greetUser`): The name of the recipe.
  • Parentheses `()`: Can contain parameters (ingredients).
  • Curly braces `{}`: Enclose the function body (the steps in the recipe).

`event` → Like ringing a doorbell — you decide what happens next!

  • An action that occurs in the browser (e.g., click, mouseover).
  • `onclick`: An event listener, waiting for a 'click' event.
  • When the event happens, the associated JavaScript code runs.
<button onclick="greetUser()">Say Hello</button>
<p id="greeting"></p>
<script>
  function greetUser() {
    document.getElementById("greeting").innerText = "Hello, lovely human!";
  }
</script>

When you click the "Say Hello" button, the browser finds the paragraph with the ID "greeting" and changes the text inside it. This is the power of JavaScript – it adds *behavior* and *interactivity*.

Statements

A **statement** is a single instruction you give to the browser. It's a command that tells the browser what to do.

alert("Thank you."); // This tells the browser to pop up a box.

The **semicolon (`;`)** at the end of the statement tells JavaScript that it's the end of the command, just as a period ends a sentence. While JavaScript can sometimes infer the end of a statement with a line break, it's best practice to always end each statement with a semicolon for clarity and to prevent potential issues.

Comments

**Comments** are notes you leave in your code that are completely ignored by the browser. They are for humans to understand the code, especially helpful if someone else (or your future self!) needs to read or edit it.

  • Single-line comments: Use two slash characters (`//`) at the beginning of the line. Everything after `//` on that line is a comment.
  • Multi-line comments: Use `/*` to start the comment and `*/` to end it. Everything between these characters is ignored, even across multiple lines.
// This is a single-line comment.
alert("Hello!"); // This comment explains what the line does.

/*
This is a multiline comment.
Anything between these characters
will be completely ignored.
*/

Variables

A **variable** is like a labelled box where you store information. You give it a name and assign it a value (which can be a number, text, a webpage element, or even a function). You can then refer to that value by its name and even change it later.

To create a variable and assign a value:

let myNumber = 5; // 'let' is the modern way to declare variables.

You can also declare a variable without an initial value:

let myText; // This variable currently holds 'undefined'.

Variable Naming Rules:

  • Must start with a letter or an underscore (`_`).
  • Can contain letters, digits, and underscores.
  • May not contain spaces (use `my_variable` or `myVariable` - **camelCase**).
  • May not contain special characters (e.g., `!`, `.`, `?`).
  • JavaScript is **case-sensitive** (`myVariable` is different from `MyVariable`).

Choose descriptive names (e.g., `productName` instead of `data`).

Data Types

The values you assign to variables fall under different **data types**. Understanding these types helps you know how JavaScript will treat your data.

  • `Undefined`: A variable declared but not yet assigned a value. 
    let foo;
alert(foo); // Shows "undefined"
  • `Null`: Represents the intentional absence of any object value. 
    let foo = null;
alert(foo); // Shows "null"
  • `Number`: For numeric values (integers and decimals). 
    let foo = 5;
alert(foo); // Shows "5"
alert(foo + foo); // Shows "10" (mathematical addition)
  • `String`: For text. Enclosed in single (`'...'`) or double (`"..."`) quotes. 
    let foo = "five";
alert(foo); // Shows "five"
alert(foo + foo); // Shows "fivefive" (string concatenation)
alert("5" + "5"); // Shows "55" (numbers treated as strings)
  • `Boolean`: Represents `true` or `false`. Used for logical conditions. 
    let isLogged = true; // 'isLogged' is now true

    Values like `null`, `undefined`, `0`, and empty strings (`""`) are considered "falsy," while others are "truthy."

  • `Array`: A collection of multiple values (called members) assigned to a single variable. Values are accessed by their **index** (position), starting from `0`. Enclosed in square brackets `[]`. 
    let colors = ["red", "green", "blue"];
alert(colors[0]); // Shows "red"
alert(colors[1]); // Shows "green"

Operators

Operators perform operations on values and variables. They are essential for calculations and making comparisons.

  • `+` (Addition)
  • `-` (Subtraction)
  • `*` (Multiplication)
  • `/` (Division)
  • `++` (Increment by 1, e.g., `x++`)
  • `--` (Decrement by 1, e.g., `x--`)
  • `+=` (Add and assign, e.g., `x += 5` is `x = x + 5`)

These operators compare values and return a `Boolean` (`true` or `false`).

  • `==` (Equal to - checks value, may convert types)
  • `!=` (Not equal to)
  • `===` (Identical to - checks value AND data type)
  • `!==` (Not identical to)
  • `>` (Greater than)
  • `>=` (Greater than or equal to)
  • `<` (Less than)
  • `<=` (Less than or equal to)

`==` vs `===` (Crucial Difference):

alert(5 == "5");   // true (values are same, "5" is converted to number)
alert(5 === "5");  // false (values are same, but types are different: Number vs String)
alert(5 !== "5");  // true (types are different, so they are not identical)

Warning: Be careful not to accidentally use a single equals sign (`=`) when you mean to compare! `=` is for *assigning* a value.

If/Else Statements

`if/else` statements are how JavaScript makes decisions. They execute different blocks of code based on whether a condition is `true` or `false`.

Basic Structure:

if (condition is true) {
    // Do this code
} else {
    // Otherwise, do this code
}

Example:

let score = 85;
if (score >= 70) {
    alert("You passed!");
} else {
    alert("You need to study more.");
}

The code inside the `if` block runs if the condition is `true`. Otherwise, the code inside the `else` block runs.

For Loops

`for` loops are used to repeat a block of code a specific number of times, or for each item in a list (like an array).

Basic Structure:

for (initialization; condition_to_continue; step_each_time) {
    // Code to repeat
}

Example (counting):

for (let i = 0; i < 3; i++) { // Starts i at 0, continues as long as i < 3, adds 1 to i each time
    console.log("Count:", i); // Will print "Count: 0", "Count: 1", "Count: 2"
}

Example (looping through an array):

let fruits = ["apple", "banana", "cherry"];
for (let i = 0; i < fruits.length; i++) { // fruits.length gives the number of items in the array
    console.log("Fruit:", fruits[i]); // Accesses item at current index 'i'
}

Events

An **event** is something that happens in the web browser (e.g., a user click, mouse movement, key press, or page loading). JavaScript lets you "listen" for these events and run specific code in response.

The `onclick` attribute in HTML is a simple way to attach an event. More advanced methods involve JavaScript's `addEventListener`.

<button onclick="myFunction()">Click Me</button>

This tells the browser: "WHEN this button is clicked (`onclick`), THEN run `myFunction()`."

The Browser Object

The `window` object is like the main control panel for your web browser. It gives JavaScript access to everything related to the browser window and the webpage it's showing.

  • `window`: Represents the browser window itself. It has properties (like `window.innerWidth` for width) and methods (like `window.open()` to open new tabs).
  • `document`: A key object within `window`. It represents the actual HTML webpage loaded. This is your primary way to interact with the HTML elements.
console.log(window.innerWidth); // Get browser window width
document.body.style.backgroundColor = "lightblue"; // Change page background

DOM: The Document Object Model

The **DOM** is like a tree-like map of your HTML structure. Every HTML element (`

`) is a "node" on this tree. JavaScript uses this map to find, change, add, or remove HTML elements and their styles.

JavaScript uses the `document` object to interact with the DOM:

  • Find elements: `document.getElementById("someId")`
  • Change content: `element.textContent = "New text";`
  • Change styles: `element.style.color = "red";`
  • Add new elements: `document.createElement("p")`, `parentElement.appendChild(newElement)`
let myDiv = document.getElementById("myDiv");
myDiv.textContent = "Content changed by JS!";
myDiv.style.backgroundColor = "yellow";

Polyfills

A **polyfill** is a piece of JavaScript code that provides the functionality of a newer JavaScript feature in older web browsers that don't natively support it. Think of it as a translator that helps older browsers understand modern code.

They ensure your website works consistently across a wider range of browsers, even those that might be outdated or lack certain modern capabilities.

JavaScript Libraries

A **JavaScript library** is a collection of pre-written JavaScript code that provides ready-made tools (functions) to help you with common tasks, simplifying web development.

  • Shorter syntax: Do more with less code.
  • Simplified tasks: Make complex things (like DOM manipulation, animations, Ajax requests) easier.
  • Cross-browser compatibility: Often handle browser differences for you.

Example: jQuery simplifies finding elements, handling events, and making server requests.

Key Code Examples & Functions

Here's a quick reference to essential JavaScript snippets and functions we've explored, which are fundamental building blocks for web interactivity.

  • Displaying messages: `alert("Your message");`, `console.log("Your message");`
  • Getting input: `document.getElementById('inputId').value;`
  • Displaying output on page: `document.getElementById('outputId').textContent = "Your message";`
  • Converting to number: `Number(someVariable);`
  • Checking if NOT a number: `isNaN(someVariable);`
  • Checking string content: `someString.includes('text');`
